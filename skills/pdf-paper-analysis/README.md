# PDF Paper Analysis Skill

这个 skill 用于把 PDF 论文转成可直接支持研发落地的技术文档。

它的目标不是做泛泛的论文摘要，而是围绕以下产出展开：

- 抓取 PDF 文献中的可读内容
- 提炼问题定义、建模假设和关键变量
- 推导核心算法与实现原理
- 编写原理分析文档
- 编写开发指导手册

## 适用场景

适合以下类型的请求：

- 分析一篇 PDF 论文
- 提炼论文中的核心算法
- 从文献中归纳实现原理
- 把研究论文整理成开发说明
- 生成原理文档和开发指导手册

尤其适合以下资料：

- 电磁建模论文
- 信号完整性论文
- 数值计算与优化论文
- 物理建模与等效电路论文
- 含公式、模态分解、参数扫描、验证曲线的研究文章

## 默认输出

默认会生成两份文档：

1. `<pdf-stem>-principle-analysis.md`
2. `<pdf-stem>-development-guide.md`

如果用户明确要求只输出一个文件，可以将两部分合并。

## 工作流程

### 1. 识别目标 PDF

- 扫描工作区中的 PDF 文件
- 如果只有一个 PDF，直接使用
- 如果有多个 PDF 且目标不明确，要求用户指定

### 2. 提取正文内容

- 先配置 Python 环境
- 优先使用 `pypdf` 提取文本
- 按页提取，保留页码线索
- 如果公式提取损坏，结合上下文、图题和参考文献还原算法结构

### 3. 建立证据映射

至少提取以下信息：

- 标题、作者、年份、出处
- 问题定义
- 核心贡献
- 建模假设
- 关键变量与符号
- 公式组或算法流程
- 验证方式
- 参数扫描或优化变量
- 主要结论

### 4. 推导算法与实现原理

把论文内容整理成可实现的结构，重点回答：

- 在解什么问题
- 输入输出是什么
- 模型由哪些子模块组成
- 哪些项来自解析推导，哪些来自数值求解
- 精度和带宽由哪些机制决定
- 软件实现时该如何拆模块和组装求解流程

### 5. 产出文档

- 原理分析文档：解释方法为什么成立、关键公式链路是什么、假设边界在哪里
- 开发指导手册：解释如何落地实现、如何组织模块、如何验证和优化

## 模板说明

- `templates/principle-analysis-template.md`
  用于约束原理分析文档结构，强调问题定义、建模假设、公式链路、误差来源和工程解释。

- `templates/development-guide-template.md`
  用于约束开发指导手册结构，强调软件架构、数据模型、求解流程、数值问题、验证与优化。

## 使用示例

可以直接用类似下面的请求触发：

- 分析这篇 PDF 论文，提炼核心算法并写开发指导手册
- 读取工作区中的 PDF 文献，推导实现原理并输出两份技术文档
- 从这篇论文中整理建模流程、关键公式和软件实现方案
- 抓取 PDF 文章内容，生成原理分析和开发说明

## 质量要求

该 skill 的输出应满足以下标准：

- 不是简单摘要，而是工程化拆解
- 能区分论文事实与工程推断
- 能明确指出公式不确定项
- 不长段复制论文原文
- 输出足够具体，开发者可以据此开始编码

## 已知限制

- 如果 PDF 是扫描版且缺少 OCR 工具，公式恢复能力会明显下降
- 如果原文公式提取损坏，精确闭式表达仍需回到原论文或其参考文献核对
- 对高度经验化、几乎不含推导的论文，最终输出会更偏工程归纳而不是严格数学复现

## 建议配套做法

实际使用时，建议将该 skill 与以下动作结合：

- 保存论文 PDF 到工作区
- 在输出文档中标记需要二次核对的公式
- 对关键算法再补读论文引用的基础文献
- 如需编码，基于开发指导手册继续生成原型求解器
